## 7. How to run / gotchas (hard-won this session)

- **Board runs are one-at-a-time and human-in-the-loop.** The board (`ivan@aifoundry2`) needs your Tailscale/SSH auth. **Do NOT stack retries** — this session lost ~1h to 4 stacked `run_smolvlm2_video_benchmark` chains fighting over the single `/dev/et0_ops` (→ `Device or resource busy`, server crashes, `float(None)` scorer aborts). Run one, let it finish.
  - Launch: `bash local-artifacts/run_when_free.sh local-artifacts/run_smolvlm2_cont_candidate_board.sh` (waits for the device, builds from the working tree, scores).
  - **That script is candidate-only** (compares to a hardcoded M0 baseline). For a real keep/drop, run it once per side (candidate, then main) or use the `smolvlm2-ab-score` skill (interleaved, drift-controlled).
  - Validate a run is real: `scored_at` is fresh and there are no `busy` / `Remote end closed` / `rc=-6` / `missing score` lines. A single run's noise floor is ~0.15% (two no-change mains differed by 0.14%), so **treat any delta <~0.5% as noise** and require the ≥1% gate.
  - If it fails repeatedly: check `ssh ivan@aifoundry2 'fuser -v /dev/et0_ops; pgrep -af run_smolvlm2_video_benchmark'` for stacked/foreign jobs before re-firing.
- **Local math gate (no board):** `smolvlm2-build-verify` — build one kernel ELF, `test-backend-ops -b ET -o <OP> -p type_a=f32,type_b=f32` (filter or it times out under sys-emu), require PASS. `ET_PLATFORM=~/et-src/et` is needed for the host build.
- **WASH discipline:** a change that doesn't clear ≥1% is reverted and logged in `EXPERIMENTS.md` with WHY, never retried. This is how the ladder stays honest.
